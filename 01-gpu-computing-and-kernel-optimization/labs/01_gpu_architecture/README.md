# Lab 01: GPU 架构查询

## 实验目的

通过查询 H20 GPU 的硬件参数，理解每个指标的工程含义。不是简单地打印数字，而是要能解释"这个数字意味着什么，对我的 AI 推理/训练有什么影响"。

## 前置要求

- Python 3.8+
- NVIDIA GPU（H20 或其他）
- `pynvml` 库：`pip install pynvml`
- `torch`（可选，用于交叉验证）

## 实验步骤

1. 运行 `query_device.py`，观察输出
2. 对照 theory/01 中的架构图，理解每个参数
3. 思考以下问题：
   - H20 最多能同时运行多少个线程？
   - 显存带宽 4 TB/s 意味着什么？
   - NVLink 拓扑如何影响并行策略的选择？

## 关键概念

- **SM 数量**：决定 kernel 能用多少并行度
- **CUDA Core / Tensor Core 数量**：决定算力上限
- **显存容量与带宽**：决定能放多大模型、数据喂得多快
- **NVLink 拓扑**：决定多卡通信效率

## 预期输出

运行脚本后将看到硬件参数 + 工程解读。关注那些影响日常工作的指标。

## 延伸练习

1. 修改脚本，计算 GPU 的 Roofline 平衡点
2. 比较 8 张 GPU 的参数是否一致
3. 在不同负载下观察 GPU 利用率变化
