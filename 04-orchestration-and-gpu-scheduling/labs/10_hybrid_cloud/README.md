# Lab 10 - 混合云 GPU 调度

## 目标

设计一个混合云 GPU 调度方案：私有集群 (8x H20) 为主，
云上 GPU 按需 burst，通过成本优化器做智能调度决策。

## 背景

```
混合云 GPU 调度的核心挑战：

1. 成本优化
   私有 H20: 固定成本（已购入）→ 边际成本≈0（电费）
   云上 GPU: 按用量计费 → 需要精确控制使用时间

2. 数据引力
   训练数据存储在私有 IDC → 传到云上耗时/费钱
   → 优先在数据所在地训练

3. 网络延迟
   跨集群通信延迟高 → 不适合跨集群分布式训练
   → 每个集群内独立训练，集群间做任务级调度

4. GPU 异构性
   私有: H20 (96GB HBM3, 适合推理)
   云 A: A100 (80GB, 通用)
   云 B: H100 (80GB, 训练最强)
   → 不同任务匹配不同 GPU 类型
```

## 决策框架

```
收到 GPU 任务请求
    │
    ▼
┌─────────────────────────────────┐
│ 私有集群有足够空闲 GPU？         │
│   → 是：调度到私有集群           │──── 成本最低
│   → 否：继续                    │
└────────────────┬────────────────┘
                 ▼
┌─────────────────────────────────┐
│ 任务优先级高？                   │
│   → 是：立即 burst 到云上       │──── 保证 SLA
│   → 否：继续                    │
└────────────────┬────────────────┘
                 ▼
┌─────────────────────────────────┐
│ 预计等待时间 < 阈值？            │
│   → 是：排队等待私有集群         │──── 节省成本
│   → 否：burst 到云上（选最便宜的）│
└─────────────────────────────────┘
```

## 思考题

1. 如何计算"burst 到云上"的经济阈值？（等多久才值得用云 GPU？）
2. 数据传输成本如何纳入调度决策？
3. 如何处理"云上训练完成后模型拉回本地"的场景？
