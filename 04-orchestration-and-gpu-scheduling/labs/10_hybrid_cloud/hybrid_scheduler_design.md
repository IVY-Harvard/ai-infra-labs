## 2. 调度决策引擎

### 2.1 输入参数

```yaml
# 调度决策输入
task:
  name: "llama-70b-finetune"
  gpu_count: 8
  gpu_memory_min: 80GB
  estimated_duration: 24h
  data_location: onprem        # 数据在私有集群
  priority: high
  deadline: "2024-12-20T00:00:00Z"
  
constraints:
  max_cost: $500
  data_transfer_allowed: true
  gpu_types_allowed: [H20, A100, H100]
  
preferences:
  prefer_onprem: true
  spot_acceptable: true        # 可以用 Spot
  cross_region: false          # 不跨区域
```

### 2.2 决策维度

```
决策维度与权重：

维度            | 权重 | 计算方式
--------------+------+----------------------------
成本           | 40%  | GPU 时间费用 + 数据传输费用
数据亲和性     | 25%  | 数据在哪个集群
等待时间       | 20%  | 预计多久可以开始训练
GPU 匹配度    | 10%  | GPU 型号与任务的匹配程度
风险           | 5%   | Spot 中断概率

计算公式：
  score(cluster) = Σ(weight_i × normalized_metric_i)
  
选择 score 最高的集群
```

### 2.3 Burst 策略

```
何时 Burst 到云上：

条件 1（紧急任务）：
  IF task.priority == "critical" AND onprem.available_gpus < task.gpu_count:
    → 立即 burst（不考虑成本）

条件 2（经济划算）：
  wait_time = estimate_wait_time(onprem_queue)
  burst_cost = cloud_gpu_price * task.gpu_count * task.duration
  wait_cost = opportunity_cost_per_gpu_hour * task.gpu_count * wait_time
  
  IF burst_cost < wait_cost:
    → burst 到云上

条件 3（deadline 约束）：
  IF task.deadline - now < task.estimated_duration + safety_margin:
    → 必须 burst（否则会 miss deadline）
```

## 3. 数据同步策略

```
┌────────────────────────────────────────────────────┐
│              数据同步决策                            │
│                                                     │
│  数据量    < 100GB    → 实时同步（开始训练前传输）   │
│  数据量    100GB-1TB  → 预热同步（提前开始传输）    │
│  数据量    > 1TB      → 只在数据所在集群训练        │
│                                                     │
│  同步方式：                                         │
│  - S3/GCS 对象存储同步                              │
│  - Rclone 增量同步                                  │
│  - Dataset Caching（云上缓存常用数据集）            │
└────────────────────────────────────────────────────┘
```

## 4. 网络互通方案

```
私有集群 ←→ 云集群 网络连接：

方案 1：VPN 隧道
  优点：简单，成本低
  缺点：带宽有限（~1Gbps），延迟高
  适合：管理通信、小数据量

方案 2：专线（Direct Connect / Interconnect）
  优点：高带宽（10-100Gbps），低延迟
  缺点：成本高，建设周期长
  适合：频繁大量数据传输

方案 3：Overlay 网络（如 Submariner）
  优点：Pod 级互通，对应用透明
  缺点：性能开销
  适合：需要跨集群 Service 发现

你的建议方案：
  - 管理面：VPN（K8s API 通信）
  - 数据面：对象存储同步（训练数据）
  - 不做跨集群分布式训练（延迟太高）
```

## 5. 对你的 8x H20 环境

```
阶段 1（当前）：
  - 只用私有集群
  - Kueue 管理队列和配额
  - 充分利用 8 张 H20

阶段 2（GPU 不够用时）：
  - 接入 1 个云 GPU 集群
  - 部署成本优化器（cost_optimizer.py）
  - 策略：优先私有，高优先级/紧急任务 burst 到云

阶段 3（规模化）：
  - 多云多集群
  - Karmada 统一管理
  - 自动化成本控制和容量规划
```
