# Lab 08: GPU 推理成本核算与计费

## 概述

GPU 推理成本管理解决三个核心问题:
1. **归因**: 每个业务线/用户消耗了多少 GPU 资源?
2. **计费**: 如何公平地分摊 GPU 成本?
3. **优化**: 哪些资源被浪费了? 如何降本?

## 成本结构

```
GPU 推理 TCO (Total Cost of Ownership)
├── 40% 硬件折旧 (GPU + Server + NVSwitch)
│   ├── H20 GPU: ~$15,000 × 8 = $120,000/节点
│   ├── Server (CPU/RAM/NVMe): ~$30,000
│   └── 3 年折旧: $150,000 / 36 months = $4,167/月
├── 25% 电力 (含 PUE)
│   ├── 8×H20 TDP: 3,200W + Server: 800W = 4,000W
│   ├── PUE 1.3: 5,200W 实际
│   └── $0.08/kWh × 5.2kW × 730h = $304/月
├── 20% 人力 (运维/MLOps)
├── 10% 网络与存储
└── 5% 其他 (冷却/机房/保险)
```

## 计费模型对比

| 模型 | 优点 | 缺点 | 适用 |
|------|------|------|------|
| GPU-Hour | 简单直观 | 不考虑利用率 | 独占式 |
| Token 计费 | 按使用量付费 | 需要精确计量 | 共享式 |
| 混合 (保底+超出) | 平衡成本与灵活性 | 复杂 | 企业内部 |
| QoS 分级 | 差异化服务 | 定价复杂 | 多 SLA 场景 |
