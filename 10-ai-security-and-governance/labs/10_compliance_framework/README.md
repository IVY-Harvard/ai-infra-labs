# Lab 10: AI Compliance Framework

## Objectives
- Understand the four-layer AI governance model (Policy / Process / Tools / Audit)
- Implement a risk assessment tool for AI models
- Build a rule-based policy engine for compliance checks

## Prerequisites
- Python 3.9+
- Basic understanding of AI/ML model lifecycle

## Lab Structure

| File | Description |
|------|-------------|
| `governance_template.md` | Four-layer governance framework template |
| `risk_assessment.py` | AI model risk scoring tool |
| `policy_engine.py` | Rule-based policy execution engine |

## Exercises

### Exercise 1: Risk Assessment
Run the risk assessment tool against sample model metadata:
```bash
python risk_assessment.py
```
Review the output risk levels and understand how scoring works.

### Exercise 2: Policy Engine
Execute the policy engine with sample rules:
```bash
python policy_engine.py
```
Try adding custom rules and observe compliance results.

### Exercise 3: Governance Template
Review `governance_template.md` and fill in sections relevant to your organization.

## Key Concepts
- **Policy Layer**: High-level principles and standards
- **Process Layer**: Workflows and procedures for compliance
- **Tools Layer**: Technical controls and automation
- **Audit Layer**: Monitoring, logging, and review mechanisms
